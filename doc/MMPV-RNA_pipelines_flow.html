🔄 双模式提交策略

suvtk 模式：source.src → suvtk comments → table2asn → .sqn（自动化程度高）
Sequin 模式：.fsa + .tbl + .cmt → tbl2asn → .sqn（手动可控，Cenote-Taker3 风格，作为 suvtk 失败时的回退方案）

📋 v2.3 → v3.0 主要变更

  • 版本号：v2.3 → v3.0，README 同步更新
  • BBNorm 阶段：新增 00c BBNorm 归一化阶段，共组装前可选 k-mer 覆盖度均衡
  • 宿主去除升级：从仅 Bowtie2 扩展为三引擎可选（Bowtie2 / HISAT2 / Minimap2），支持多种测序类型
  • 组装后处理：新增 refineC split → merge 后处理流程，拆分嵌合 contig
  • 鉴定工具数：9 → 10，新增 Viroid BLASTN 专门捕获类病毒序列
  • 聚类输出重构：centroids 提升为独立目录，known/novel contig 分流更清晰
  • 分类工具数：9 → 8，优化工具组合，R 共识引擎升级
  • 拯救分支扩展：3 支路 → 4 支路，新增属水平长度拯救（同属 ±15%）作为兜底策略
  • 宿主预测重构：RNAVirHost + iPHoP → RNAVirHost + PhaBOX2/CHERRY，决策树逻辑优化
  • 提交管线独立：virome_submission_pipeline 从附属脚本提升为独立管道，增加 suvtk 模式和 GUI
  • 环境管理：pixi.toml 新增 feature 分组（discovery / analysis / publicdata / gpu），按需安装
